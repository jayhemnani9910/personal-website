# Interview Prep Template

## Questions to Ask Them
1. "What does the tech stack look like? What's the biggest technical challenge right now?"
2. "How is the team structured? Who would I work closely with?"
3. "What does success look like in the first 90 days for this role?"
4. "What's the team's approach to code review and shipping?"
5. "How do you handle on-call / incidents?"

## Common Questions — Prepared Answers

### Tell me about yourself
"I'm a CS grad from PDEU with hands-on experience in data engineering and ML. I've worked as a Data Analyst at Elite Hotel Group automating reporting pipelines, and as an AI/ML Intern at Amnex building fraud detection systems. I've published 2 IEEE papers and built 25+ projects — from FAISS vector search engines to computer vision pipelines. I ship fast and I'm looking for a startup where I can own problems end-to-end."

### Why should we hire you?
"Three things: I ship fast — 25+ projects, 2 IEEE papers, and I implemented a brand-new W3C standard within days of its release. I'm versatile — Python, SQL, PyTorch, Docker, K8s, Go, TypeScript. And I thrive in fast-paced environments — my best work has been at startups and small teams where I can own the full stack."

### What's your biggest weakness?
"I sometimes over-optimize early. I've learned to ship the MVP first and iterate based on real feedback rather than engineering for hypothetical scale."

### Tell me about a challenging project
"BarçaBrain — I built a football intelligence platform with FAISS vector search for player similarity. The challenge was feature engineering on messy match event data into meaningful dense embeddings. I had to balance retrieval speed (<100ms target) with result quality, using hybrid k-NN filtering. The result was a system that scouts could actually use for rapid player discovery."

### Where do you see yourself in 5 years?
"Leading a data/ML team at a company whose product I believe in. I want to go deep technically while growing into someone who can mentor others and shape technical direction."

- System design: data pipelines, ML serving, API design
- Coding: Python (data structures, algorithms), SQL (joins, window functions, CTEs)
- ML: model selection, evaluation metrics, feature engineering, deployment
- Data: ETL patterns, data quality, schema design, query optimization
