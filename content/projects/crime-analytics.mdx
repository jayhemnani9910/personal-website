  context: |
    Crime data analysis plays a critical role in evidence-based public safety policy and resource allocation. By analyzing historical incident patterns, law enforcement agencies can identify trends, anticipate seasonal variations, and deploy resources more effectively to areas of greatest need.

    This project focuses on Indian crime data, addressing the challenge of fragmented datasets from multiple sources with inconsistent taxonomies. The goal is to create a unified analytical framework that enables both exploratory analysis and predictive modeling for incident forecasting.

    Understanding spatial and temporal patterns in crime data helps communities make informed decisions about safety initiatives, while respecting the sensitivity of the subject matter and maintaining statistical rigor.

  keyDecisions:
    - decision: "Unified Incident Taxonomy"
      rationale: "Different data sources classified crimes differently (e.g., 'theft' vs 'robbery' vs 'burglary'). Created hierarchical taxonomy mapping specific incidents to broader categories for consistent analysis while preserving original classifications."

    - decision: "Time-Series Aggregation Strategy"
      rationale: "Chose daily aggregation level as baseline (vs hourly/weekly) to balance granularity with statistical stability. Hourly data too sparse for many categories; weekly aggregation obscured day-of-week patterns critical for resource planning."

    - decision: "Spatial Clustering with DBSCAN"
      rationale: "Selected DBSCAN over K-means for hotspot detection because it handles arbitrary-shaped clusters and doesn't require pre-specifying cluster count. Critical for identifying natural crime concentration areas rather than forcing geometric partitions."

    - decision: "Missing Data Handling Approach"
      rationale: "Used domain-specific imputation strategies: forward-fill for sequential reports, median imputation for numeric fields, and 'Unknown' category for missing classifications. Avoided deletion to preserve temporal continuity and sample size."

    - decision: "Prophet for Forecasting Framework"
      rationale: "Facebook Prophet chosen over traditional ARIMA because it handles multiple seasonality patterns (daily, weekly, yearly) naturally and is robust to missing data. Particularly effective for capturing festival/holiday effects in Indian context."

        import numpy as np
        from sklearn.cluster import DBSCAN
        from sklearn.preprocessing import StandardScaler

        def identify_crime_hotspots(df, eps_km=0.5, min_samples=10):
            """
            Detect geographic clusters of incidents using DBSCAN.

            Args:
                df: DataFrame with 'latitude' and 'longitude' columns
                eps_km: Maximum distance (km) between points in cluster
                min_samples: Minimum incidents to form a hotspot

            Returns:
                DataFrame with cluster labels and centroids
            """
            # Extract coordinates
            coords = df[['latitude', 'longitude']].values

            # Convert eps from km to degrees (approximate)
            # 1 degree ≈ 111 km at equator
            eps_degrees = eps_km / 111.0

            # Cluster using DBSCAN
            clustering = DBSCAN(
                eps=eps_degrees,
                min_samples=min_samples,
                metric='haversine'
            ).fit(np.radians(coords))

            df['hotspot_cluster'] = clustering.labels_

            # Calculate cluster centroids (excluding noise: label=-1)
            hotspots = df[df['hotspot_cluster'] != -1].groupby('hotspot_cluster').agg({
                'latitude': 'mean',
                'longitude': 'mean',
                'incident_id': 'count'
            }).rename(columns={'incident_id': 'incident_count'})

            return df, hotspots

    - label: "Time-Series Seasonal Decomposition"
      code: |
        import pandas as pd
        from statsmodels.tsa.seasonal import seasonal_decompose
        import matplotlib.pyplot as plt

        def analyze_temporal_patterns(df, category='All', period=365):
            """
            Decompose incident time-series into trend, seasonal, residual.

            Args:
                df: DataFrame with 'date' and 'incident_count' columns
                category: Incident category to analyze
                period: Seasonal period (365 for yearly pattern)
            """
            # Filter and aggregate daily counts
            if category != 'All':
                df = df[df['category'] == category]

            daily_counts = df.groupby('date').size().reset_index(name='count')
            daily_counts['date'] = pd.to_datetime(daily_counts['date'])
            daily_counts = daily_counts.set_index('date').asfreq('D', fill_value=0)

            # Decompose time-series
            decomposition = seasonal_decompose(
                daily_counts['count'],
                model='additive',
                period=period
            )

            # Plot components
            fig, axes = plt.subplots(4, 1, figsize=(12, 10))
            daily_counts['count'].plot(ax=axes[0], title='Original')
            decomposition.trend.plot(ax=axes[1], title='Trend')
            decomposition.seasonal.plot(ax=axes[2], title='Seasonality')
            decomposition.resid.plot(ax=axes[3], title='Residuals')
            plt.tight_layout()

            return decomposition

    - label: "Multi-Category Correlation Analysis"
      code: |
        import pandas as pd
        import seaborn as sns
        import matplotlib.pyplot as plt

        def analyze_category_correlations(df, categories=None, freq='M'):
            """
            Compute and visualize correlations between incident categories.

            Args:
                df: DataFrame with 'date' and 'category' columns
                categories: List of categories to analyze (None = all)
                freq: Aggregation frequency ('D'=daily, 'W'=weekly, 'M'=monthly)

            Returns:
                Correlation matrix DataFrame
            """
            # Pivot to get category counts by time period
            df['date'] = pd.to_datetime(df['date'])
            pivot = df.groupby([pd.Grouper(key='date', freq=freq), 'category']).size()
            pivot = pivot.unstack(fill_value=0)

            if categories:
                pivot = pivot[categories]

            # Calculate correlation matrix
            corr_matrix = pivot.corr()

            # Visualize
            plt.figure(figsize=(10, 8))
            sns.heatmap(
                corr_matrix,
                annot=True,
                fmt='.2f',
                cmap='coolwarm',
                center=0,
                vmin=-1,
                vmax=1,
                square=True
            )
            plt.title(f'Incident Category Correlations ({freq} aggregation)')
            plt.tight_layout()

            return corr_matrix

  learnings:
    - insight: "Data Quality Trumps Algorithm Complexity"
      detail: "The biggest impact came from careful data cleaning and taxonomy normalization, not sophisticated models. Spending 60% of time on preprocessing (deduplication, geocoding validation, category standardization) yielded far better insights than rushing to analysis with messy data."

    - insight: "Geographic Context Is Critical"
      detail: "Raw latitude/longitude alone provided limited value. Enriching data with district boundaries, population density, economic indicators, and distance to landmarks revealed patterns invisible in pure spatial clustering. Domain knowledge about regional characteristics proved essential."

    - insight: "Multiple Seasonality Patterns Exist"
      detail: "Crime data exhibited complex temporal patterns: day-of-week effects (weekends vs weekdays), monthly variations (summer vs monsoon), and festival-related spikes. Single-seasonality models failed; multi-seasonal approaches (Prophet) captured reality better."

    - insight: "Aggregation Level Affects Interpretability"
      detail: "Hourly data appeared too noisy for forecasting but revealed important patterns (late night spikes). Monthly aggregation smoothed trends but obscured weekly cycles. Maintaining multiple temporal resolutions for different analytical questions proved most effective."

    - insight: "Correlation Doesn't Imply Causation in Crime Data"
      detail: "Found strong correlations between certain incident categories, but careful interpretation required. Example: theft and vandalism correlated due to shared underlying factors (economic conditions, policing levels) rather than direct causation. Statistical relationships needed contextual validation."

    - insight: "Ethical Considerations in Presentation"
      detail: "Learned to present findings responsibly: avoiding stigmatization of specific communities, acknowledging data collection biases (underreporting varies by category/region), and framing insights around resource optimization rather than punitive measures.
